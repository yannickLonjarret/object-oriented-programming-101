# Object Oriented Programming 101
This repository contains the rewriten source code of an assignment developed as part of a **Object Oriented Programmin** course at **UTC** during the **Spring 2023** semester.

While the project is simple, the work here was an opportunity to learn CMake, Qt, and idiomatic C++ (RAII, STL) by applying real-world software engineering practices to that material.
## The project

The project was to be completed during the semester by groups of four to five. In addition to the source code, students were required to write three intermediary reports during the semester and a final report.

To preserve the confidentiality of the original academic material, the following instructions are descriptions based on the original, rather than exact reproductions.

### Schotten Totten

The project was to design and develop a **C++** application which allowed us to play the **Schotten Totten** card game. 

**Schotten Totten** is a 2-player card game where players compete to win "stones" (or territories) on a battlefield. By forming poker-like combinations (e.g., runs, flushes) on each stone, players try to outscore their opponents and secure the majority of stones.

The project requirement were as follows:
- Every functionalities of the original game were to be implemented
- Each players could be an "AI"
- The game must be playable at least using a CLI, at best a Qt GUI
- Finally, the application architecture should be easily extensible, allowing a selection of new functionalities to be added with minimal refactoring.
